# Teleman v1.1.7: Performance, Security & Test Hardening

This release focuses on critical security patching, performance optimizations in core operations, and complete test suite hardening to ensure maximum reliability.

## 🌟 What's New

*   **🔒 Security & Permissions Hardening**:
    *   **Zip Slip / Path Traversal Fix**: Patched a directory traversal vulnerability during download extraction to prevent malicious overwrites of local files.
    *   **Strict Cache Permissions**: Local metadata and configuration cache permissions are now strictly locked down (files created with `0600` and directories with `0700` permissions) to prevent unauthorized read/write by other local users.
*   **⚡ Performance Optimizations**:
    *   **Index Truncation Optimization**: Resolved N+1 API request overhead when truncating indexes, significantly reducing remote round-trips.
    *   **Lazy Path Splitting**: Reduced memory allocations in `IsIgnored` checks by utilizing lazy path splitting.
    *   **Fast Hash Conversions**: Switched from slow formatting to direct `hex.EncodeToString` inside the Bolt engine, speeding up hash comparison and verification passes.
*   **🧹 Code Health & Reliability**:
    *   **Streamlined Sync Engine**: Simplified the core sync engine concurrency checker logic for more robust multi-threaded tracking.
*   **🧪 Test Hardening**:
    *   Comprehensive table-driven unit tests added for `chunkMessage`, `IsIgnored`, `ParseChunkSize`, config save/load, `HashChunk`, `DeriveKey`, `NewBarWriter`, `copyFile`, and `formatBytes`.
